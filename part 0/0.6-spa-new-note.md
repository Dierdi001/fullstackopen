sequenceDiagram
    actor User
    participant SPA as Single Page Application (Frontend)
    participant API as Backend API
    participant DB as Database

    User ->> SPA: Clicks "New Note"
    SPA ->> User: Displays note creation form

    User ->> SPA: Enters note content and submits
    SPA ->> SPA: Client-side validation

    SPA ->> API: POST /notes (note data, auth token)
    API ->> API: Validate request and authenticate user

    API ->> DB: Insert new note
    DB -->> API: Note saved successfully

    API -->> SPA: 201 Created (note metadata)
    SPA ->> SPA: Update local state (no page reload)
    SPA -->> User: Display newly created note in the UI

    Note over User,SPA: Note is created without reloading the page
